Progression of the project

  1. To start with, I started with most of the project being based off of project 2.
  2. I set out with wanting to create a site where users could upload there code files and allow them to be downloaded if set to public.
  3. Once I had the base of project 2, I deleted what I didnt need such as anything correlated to budget and journal.
  4. Once that was done, I set out to reformat the everything related to tasks to fit my needs for what I'll now call Programs.
  5. The first step to getting the site working once the unneccesary material was gone, I adjusted my settings.py and models.py for accepting a fileField.
  6. To do this, I needed to add a few lines at the bottom of settings.py and introduce fileField in my models and forms for programs with the directory set to media/documents.
  7. I ended up creating two models pertaining to my programs: Program model and Program Category model which was used within the program model.
  8. After the models were created, I then went on to adjust the views to allow for file upload.
  9. I then ran the program and tested to ensure that the file upload was working as intended.
  10. The next problem I tackled was deleting the files associated to the models.
  11. After many hours, I found that the best solution for this was to use django-cleanup, which deletes files associated to models whenever their models are deleted.
  12. Once the upload and delete were working correctly, I began to work on the download mechanism for files.
  13. This was a simple solution but took me a while to figure out. I had to add a line to urls.py and then I just used row.upload.url and download attribute in an href to make the files downloadable.
  14. After download was working properly, we then went on to adjust the edit page so that we can edit which files have been uploaded and their descriptions.
  15. I still have yet to figure it out completely, but as of now the only way to edit an input is you can't change the file linked to the model.
  16. That is a bug that I have been unable to fix unfortunately and decided to move on from for timesake.
  17. Once the edit page was done, Tommy helped me out with styling the site and finishing the about page.
  18. The styling made the site look a lot more uniform and dynamic, without being too overbearing.
  19. After the site was completed, we then dockerized it and uploaded it to GCP.
